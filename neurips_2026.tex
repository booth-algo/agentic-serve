\begin{abstract}
There is significant interest in optimising LLM serving on GPU infrastructure, driving the development of inference benchmarks across serving engines such as vLLM and SGLang. However, nearly all existing benchmarks evaluate single-turn workloads---short chat exchanges or synthetic random-token requests. In practice, the fastest-growing deployment pattern is LLM agents: coding assistants, terminal automation, and tool-use systems that issue multi-turn requests with growing context length over sessions spanning hundreds of interaction steps. No existing benchmark captures this regime. 

We introduce \textsc{AgentServe}, an agentic inference benchmark suite. Using real-life agentic traces from SWE-Bench and TerminalBench alongside standard chat baselines, we run saturation benchmarks across 3 model families on A100, H100 and RTX GPUs, measuring first-token latency, per-token latency, and output throughput at varying concurrency levels. We complement this with kernel-level Nsight Compute (ncu) traces to construct a roofline model that reveals actual GPU resource utilisation. Additionally, we train per-operator and per-kernel ML predictors on the collected profiling data, enabling latency estimations without re-running expensive GPU experiments. These data inform us on which kernels are memory-bound versus compute-bound, informing choices for prefill-decode disaggregation (PDD) and attention-FFN disaggregation (AFD) configurations for LLM inference serving of agentic systems.

\begin{abstract}
  A coding agent sends 17{,}000 input tokens per request and produces 800 output tokens---an ISL/OSL ratio of 21:1. A chat benchmark assumes 500 input and 300 output tokens, a ratio of 1.7:1. Current inference benchmarks (MLPerf, InferenceX, ShareGPT-based framework benchmarks) use the latter distribution. We show this mismatch produces wrong capacity estimates: Llama-3.1-8B on A100 sustains 920 output tokens/s under chat-short workloads but saturates at 240 tokens/s under coding-agent workloads at matched concurrency---a 3.8$\times$ gap invisible to existing benchmarks.

  We present \textsc{AgentServe-Bench}, a benchmark suite and dataset for LLM inference under agentic workloads. We contribute: (1)~a workload taxonomy with real agent traces from SWE-Bench and TerminalBench alongside ShareGPT baselines and synthetic stress tests; (2)~saturation analysis across 7 model--hardware configurations on A100 and H100, showing that agentic workloads saturate earlier and at lower throughput than chat workloads; and (3)~kernel-level roofline analysis using NVIDIA Nsight Compute hardware counters that traces this gap to a regime shift---from memory-bound decode GEMMs ($M$=1, 54\% peak HBM bandwidth) to compute-bound prefill GEMMs ($M$=512, 49\% peak TFLOPS). We release 2{,}400+ benchmark results, 1{,}900+ ncu kernel profiles, and an open-source benchmarking tool.
\end{abstract}

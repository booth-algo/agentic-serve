\begin{abstract}
  The rapid adoption of LLM-powered agents for software engineering, data analysis, and tool use has fundamentally changed inference workload characteristics. Yet the dominant benchmarking methodology---synthetic prompts with arbitrary input/output lengths in single-turn settings---fails to capture these shifts. A coding agent session averages 17,000 input tokens per request (vs.\ 500 for chat benchmarks), produces multi-turn interactions spanning hundreds of steps, and exhibits prefill-dominated compute profiles that saturate GPU throughput at dramatically different concurrency levels.

  We present \textsc{AgentServe-Bench}, a benchmarking framework and dataset for LLM inference under realistic agentic workloads. Our contributions are threefold: (1)~a four-tier workload taxonomy spanning real agent traces from SWE-Bench and TerminalBench, ShareGPT conversations, and synthetic stress tests; (2)~saturation analysis across 7 model configurations on A100 and H100 GPUs showing that agentic workloads reduce peak throughput by up to 3.8$\times$ compared to chat benchmarks at matched concurrency; and (3)~kernel-level roofline analysis using NVIDIA Nsight Compute hardware counters that explains \emph{why} agentic workloads behave differently---prefill-heavy requests shift execution from memory-bound decode kernels (54\% HBM bandwidth) to compute-bound GEMM kernels (68\% peak TFLOPS). We release 2,400+ benchmark results, 1,900+ ncu kernel profiles, and an open-source benchmarking tool to enable reproducible evaluation of inference systems under realistic agentic conditions.
\end{abstract}

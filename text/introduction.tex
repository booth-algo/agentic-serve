How should we benchmark LLM inference when the dominant workload is no longer a short chat exchange but a multi-step coding agent that sends 6{,}000+ tokens per request? LLM agents now power coding assistants, terminal automation, and computer-use systems, yet inference benchmarks continue to evaluate serving infrastructure using chat-scale traces that do not represent these workloads. These include short single-turn chat exchanges, synthetic random-token stress tests, or fixed-length request distributions drawn from datasets like ShareGPT~\citep{sharegpt}. Reddi et al.'s MLPerf Inference~\citep{mlperf,mlperf_v51} evaluates summarisation, chat, and long-context reasoning; its LLM workloads range from 1{,}024-token Q\&A (Llama~2~70B on OpenOrca) to 128K-token document tasks (Llama~3.1~405B), but remain single-turn with no multi-step tool use. The benchmarks bundled with Kwon et al.'s vLLM~\citep{vllm} default to ShareGPT prompts whose input lengths cluster around 1{,}024 tokens; Zheng et al.'s SGLang~\citep{sglang} similarly defaults to ShareGPT for serving evaluation.

Using single-turn datasets to create benchmarks leads to misleading information on actual GPU utilisation on real workloads. ShareGPT chat-short prompts have a measured ISL/OSL ratio near 1:1 (median ISL\,$\approx$\,127 tokens, OSL\,$\approx$\,151 tokens), with input lengths in the low hundreds. In contrast, a coding agent processing a SWE-Bench~\citep{swebench} task sends a repository-scale context of thousands of tokens and receives a short code patch, giving a measured ISL/OSL ratio of 19.5:1 (median ISL\,$\approx$\,6{,}236 tokens, OSL\,$\approx$\,320 tokens). This is not a quantitative difference but a qualitative regime shift: chat workloads are predominantly decode-dominated and memory-bandwidth-bound at typical serving concurrencies, while agentic workloads shift time toward compute-bound prefill GEMMs and generate sustained KV cache pressure from growing multi-turn context. Figure~\ref{fig:quadrant} maps this separation across 1{,}466 benchmark configurations.

On an H100 GPU, Llama-3.1-8B sustains 9{,}327 output tokens/s under a chat-short workload but 4{,}904 tokens/s under a coding-agent workload at saturation, a 1.9$\times$ gap in output throughput. The gap in request throughput is larger: 55.15\,req/s for chat-short versus 15.03\,req/s for coding-agent, a 3.7$\times$ reduction in serving capacity. Existing benchmarks do not sweep concurrency to saturation; MLPerf Inference reports a single operating point, and the vLLM and SGLang benchmarks default to fixed concurrency, missing the throughput ceiling and latency degradation curve that determine real-world capacity.

We present \textsc{AgentServe}, a benchmark suite and analysis framework for LLM inference under agentic workloads. Our contributions are:

\begin{enumerate}
    \item \textbf{Saturation benchmarks on agentic workloads.} We benchmark six models across three families (Llama-3, Qwen, GPT-OSS; dense and MoE, 8B--120B total parameters) on H100 GPUs using vLLM, sweeping concurrency from 1 to 320 (Section~\ref{sec:results}); A100 and SGLang comparisons are in Appendix~\ref{sec:appendix_hw}. We collect 2{,}527 benchmark configurations. Agentic workloads increase TTFT by 1.8--5.6$\times$ and reduce request throughput by up to 3.7$\times$ compared to chat baselines.

    \item \textbf{Per-kernel analysis with system-level OI/CF characterisation.} We profile ${\sim}$3{,}000 CUDA kernel invocations across 48 model configurations using PyTorch Profiler (Section~\ref{sec:kernel_roofline}). Prefill GEMMs reach arithmetic intensity up to 1{,}764 and up to 785\,TFLOPS on H100 (compute-bound); decode GEMMs operate at AI\,=\,4.0 and 2.9\,TFLOPS (memory-bound). At the system level, Tier~1 agentic and Tier~2 chat workloads separate by 56$\times$ in median operational intensity. MoE models execute 63\% more kernels per forward pass than dense models.

    \item \textbf{Per-operator latency predictors.} We train per-family XGBoost models on the collected kernel profiles, enabling latency estimation for new architectures without re-running GPU experiments (Section~\ref{sec:predictor}). On two held-out models (Qwen-2.5-72B, Llama-3.3-70B), per-kernel predictions sum to within 4.1\% mean absolute error of measured single-layer latency.
\end{enumerate}

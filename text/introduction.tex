When a coding agent processes a SWE-Bench task, it sends a 17{,}000-token repository context to the LLM and receives an 800-token code patch---an input/output ratio of 21:1. When the same model serves a chat request, the ratio is closer to 1.7:1. This 12$\times$ difference in ISL/OSL ratio changes which hardware resource is the bottleneck: chat workloads are decode-dominated and memory-bound, while agentic workloads are prefill-dominated and compute-bound. Yet every major inference benchmark---MLPerf Inference~\citep{mlperf}, InferenceX, and the ShareGPT-based benchmarks built into vLLM and SGLang~\citep{sglang}---evaluates serving systems using chat-scale request distributions.

This mismatch produces misleading capacity estimates. On an A100 GPU, Llama-3.1-8B sustains 920 output tokens/s under a chat-short workload (ISL=500, OSL=300) but only 240 tokens/s under a coding-agent workload at matched concurrency---a 3.8$\times$ gap. A practitioner sizing infrastructure from chat benchmarks would need nearly four times more GPUs than expected to meet an agentic SLO.

We present \textsc{AgentServe-Bench}, a benchmarking framework and dataset for evaluating LLM inference under agentic workloads. Our contributions:

\begin{enumerate}
    \item \textbf{Agentic workload dataset.} We extract request traces from SWE-Bench coding agents (ISL$\approx$17K, OSL$\approx$800) and TerminalBench CLI agents (multi-turn sessions spanning 2--876 steps), and pair them with ShareGPT baselines and synthetic stress tests at controlled ISL/OSL ratios (Section~\ref{sec:taxonomy}). The agent traces occupy a region of the length distribution---ISL/OSL ratios above 20:1 with input lengths exceeding 16K tokens---that no existing benchmark covers (Figure~\ref{fig:isl_osl}).

    \item \textbf{Saturation analysis.} We benchmark 6 model families (dense and MoE, 8B--120B parameters) across A100 and H100 GPUs using vLLM and SGLang, sweeping concurrency from 1 to 320 (Section~\ref{sec:results}). Agentic workloads saturate at 2--4$\times$ lower throughput than chat workloads across all configurations and exhibit steeper TTFT degradation under load.

    \item \textbf{Kernel-level roofline analysis.} Using NVIDIA Nsight Compute hardware counters, we profile 1{,}900+ CUDA kernels from Llama-8B and Mixtral-8$\times$7B forward passes (Section~\ref{sec:roofline}). The throughput gap traces to a regime shift: chat requests spend GPU time in memory-bound decode GEMMs ($M$=1, 54\% peak HBM bandwidth), while agentic requests shift time to compute-bound prefill GEMMs ($M$=512, 49\% peak TFLOPS). Mixtral's MoE architecture generates 3.7$\times$ more kernel invocations than Llama due to per-expert dispatch.
\end{enumerate}

We release 2{,}400+ benchmark result files, ncu kernel profiles, workload traces, and the benchmarking tool under an open-source license.\footnote{Dataset and code: \url{[anonymized for review]}.}

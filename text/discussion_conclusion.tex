\section{Conclusion}
\label{sec:conclusion}

We benchmark six LLMs (8B--120B parameters, dense and MoE) under Tier~1 agentic workloads from SWE-Bench and TerminalBench alongside Tier~2 ShareGPT chat baselines. Three findings emerge. First, agentic workloads increase TTFT by 1.8--5.6$\times$ in single-turn and up to 4.1$\times$ in multi-turn sessions compared to chat, while reducing single-turn request throughput by up to 3.7$\times$ and multi-turn throughput by up to 4.1$\times$ at saturation. Second, kernel-level profiling reveals the mechanism: prefill GEMMs operate at arithmetic intensity 1{,}764 (compute-bound, 785\,TFLOPS on H100), while decode GEMMs operate at arithmetic intensity 4.0 (memory-bound, 2.9\,TFLOPS). Agentic workloads shift GPU time from the memory-bound decode regime to the compute-bound prefill regime. Third, MoE architectures execute 63\% more kernels per forward pass than dense models, with elementwise operations accounting for 52.4\% of prefill time. We release 2{,}527 benchmark configurations, 420 kernel profiles, and the benchmarking tool under an open-source license to support evaluation of inference systems under realistic agentic workloads.

\paragraph{Limitations.}
(1)~We evaluate two GPU generations (A100, H100); newer hardware (B200, MI300X) may shift the compute--memory balance.
(2)~All experiments are single-node; multi-node deployments introduce communication overheads not captured here.
(3)~Kernel profiles are from 2-layer model stubs; cache effects at full depth are not measured.
(4)~We do not evaluate quantized models (FP8, INT4), which alter roofline characteristics.
(5)~Agentic traces come from software engineering agents; other agent types (web browsing, data analysis) may exhibit different ISL/OSL distributions.
(6)~Per-kernel predictor validation (Sections~\ref{sec:predictor}--\ref{sec:e2e_prediction}) is pending code audit and will be reported in a subsequent revision.

\section{Dataset and Reproducibility}
\label{sec:dataset}

We release:
\begin{itemize}
    \item \textbf{Benchmark results}: 2{,}527 JSON files with per-request TTFT, TPOT, ITL, and throughput across all (model, profile, concurrency, hardware) configurations.
    \item \textbf{Kernel profiles}: Per-kernel timing and arithmetic intensity for 420 CUDA kernels across five models at five batch sizes.
    \item \textbf{Workload traces}: Agent request traces from SWE-Bench, TerminalBench, and OSWorld, including multi-turn session logs with per-turn metrics.
    \item \textbf{Benchmark tool}: Asynchronous benchmarking framework with 16 workload profiles, configurable concurrency, supporting any OpenAI-compatible endpoint.
\end{itemize}

All artifacts are released under an open-source license with a Croissant metadata record for machine-readable discovery.\footnote{Hosted at \url{[anonymized for review]}.}

\section{Conclusion}
\label{sec:conclusion}

We benchmark six LLMs (8B--120B total parameters) under Tier~1 agentic workloads from SWE-Bench and TerminalBench alongside Tier~2 ShareGPT chat baselines. Agentic workloads increase single-turn TTFT by 1.8--5.6$\times$ and reduce request throughput by up to 3.7$\times$ at saturation. Multi-turn agentic sessions show up to 4.1$\times$ throughput reduction compared to multi-turn chat. Kernel-level profiling traces this to a regime shift: prefill GEMMs operate at arithmetic intensity 1{,}764 (compute-bound, 785\,TFLOPS on H100) while decode GEMMs operate at AI\,=\,4.0 (memory-bound, 2.9\,TFLOPS). At the system level, Tier~1 and Tier~2 workloads separate by 56$\times$ in median operational intensity. MoE architectures execute 63\% more kernels per forward pass than dense models, with elementwise operations accounting for 52.4\% of prefill time. We release 2{,}527 benchmark configurations, kernel profiles, and the benchmarking tool under an open-source license.

\section{Dataset and Reproducibility}
\label{sec:dataset}

We release:
\begin{itemize}
    \item \textbf{Benchmark results}: 2{,}527 JSON files with per-request TTFT, TPOT, ITL, and throughput across all (model, profile, concurrency, hardware) configurations.
    \item \textbf{Kernel profiles}: Per-kernel timing and arithmetic intensity for 420 CUDA kernels across five models at five batch sizes.
    \item \textbf{Workload traces}: Agent request traces from SWE-Bench, TerminalBench, and OSWorld, including multi-turn session logs with per-turn metrics.
    \item \textbf{Benchmark tool}: Asynchronous benchmarking framework with 16 workload profiles, configurable concurrency, supporting any OpenAI-compatible endpoint.
\end{itemize}

All artifacts are released under an open-source license with a Croissant metadata record for machine-readable discovery.\footnote{Hosted at \url{[anonymized for review]}.}

# Results Tracking Rules

## Where results go

- **Raw JSON**: `results/<descriptive-name>.json` (auto-generated by runner)
- **Human summary**: `.claude/docs/results.md` — always update after a notable run

## Required metadata per result entry in results.md

Every table or result block must note:
- Date
- Model + precision (e.g., Llama 3.1 70B FP8)
- Hardware (e.g., 2x H100 SXM5 80GB, RunPod)
- Server flags: prefix caching ON/OFF, chunked prefill ON/OFF, TP size
- Client flags: --ignore-eos YES/NO, arrival pattern
- OSL hit rate (output_tokens actual vs requested)
- Tool used: inference-benchmark or InferenceX

## TTFT validity annotation

Always mark TTFT as valid or invalid:
- ⚠️ TTFT (prefix cache hit) — file-based prompts + prefix caching ON
- ✓ TTFT — ShareGPT prompts (varied, no shared prefix)
- ✗ TTFT — InferenceX with --request-rate inf (includes queue wait)

## Metric reference

| Metric | Description | When valid |
|--------|-------------|------------|
| TTFT p50 | Median time to first token | Always valid with ShareGPT; see annotation for file-based |
| TPOT p50 | Median time per output token | Always valid |
| E2EL p50 | Median end-to-end latency | Always valid |
| Output tok/s | Total output tokens / benchmark duration | Always valid |
| OSL hit rate | actual_output_tokens / requested_osl | Check for FP8 + random tokens |
